AES (Advanced Encryption Standard) is a symmetric cipher — the same key encrypts and decrypts. It works in 128-bit blocks and supports 128/192/256-bit keys. We use AES-256-GCM here, which adds an authentication tag to detect any tampering.
AES-GCM Encryption Symmetric
Enter a message below to encrypt it with a randomly generated AES-256 key. The output includes the key, nonce, ciphertext, and authentication tag.
RSA is asymmetric — uses a public key to encrypt and a private key to decrypt. It can also create digital signatures: sign with private key, verify with public key. Note: RSA keys are generated in your browser here for demo purposes.
RSA Digital Signature Demo Asymmetric
Type a document and we'll simulate signing it. Then you can verify the signature, or test what happens when the document is tampered with.
SHA (Secure Hash Algorithm) produces a fixed-size fingerprint of any input. It's one-way (can't reverse it), deterministic (same input → same output), and has the avalanche effect (1 character change → ~50% of output bits change).
Hash Algorithm Comparison One-Way
See how different SHA algorithms hash the same input. Notice the output length differences and why SHA-1 is deprecated.
Algorithm Bits Status Hash Digest
Avalanche Effect Visualizer
Watch how changing a single character causes ~50% of hash bits to flip.
HMAC (Hash-based Message Authentication Code) adds a secret key to hashing. It proves both: the message wasn't tampered with AND it came from someone who knows the secret. Used in JWT tokens, webhook verification, API request signing.
HMAC-SHA256 Demo Message Auth
PBKDF2 is the secure way to store passwords. It applies HMAC-SHA256 100,000+ times with a random salt. This makes brute-force 100k× slower and rainbow tables useless. Never store passwords as plain SHA hashes!
PBKDF2 Password Hashing Secure Storage
Why Salting Matters
Two identical passwords hash to COMPLETELY different values because of random salts. This defeats rainbow table attacks.
